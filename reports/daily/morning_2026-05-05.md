## A8 Sliding Window Dataloader

`data/code_accord.py` now supports document-aware sliding windows via
`--doc-window-size` and `--doc-window-stride`. With `--doc-window-size 2`, each
sample is the current sentence plus the next same-document sentence. Entity and
relation spans are shifted by the word offset of their source sentence inside
the joined context. Relations remain sentence-local because CODE-ACCORD does
not annotate cross-sentence relation pairs.

Exact dry-run command:

```bash
PYTORCH_JIT=0 uv run python train_span.py --dataset accord \
  --model-name microsoft/deberta-base --bio-weight 0.1 \
  --neg-sample-ratio 3.0 --doc-window-size 2 --doc-window-stride 1 \
  --max-length 256 --max-steps 200 --seed 42
```

Dry-run result:

- CODE-ACCORD windows compiled: train=720, dev=126, test=173.
- Relation labels survived offsetting: train rels=2241, dev rels=391.
- Dev and test metric calculation completed without crashing.
- Final dev NER=0.4489, Triple=0.0000.
- Final test NER=0.4679, Triple=0.0000.

Triple=0 is expected for a 200-step compile smoke and should not be interpreted
as a quality comparison.

New wiki/raw note:

- `wiki/raw/KXDocRE_cross_document_RE_2024.md` — Jain et al.,
  "Knowledge-Driven Cross-Document Relation Extraction," Findings of ACL 2024,
  DOI 10.18653/v1/2024.findings-acl.227.

Key insight: longer context alone is not enough. KXDocRE suggests ranking
evidence paths between entity pairs before relation prediction. For CODE-ACCORD,
the direct Wikidata-path mechanism is not transferable, but the idea maps to
document-local paths: same `doc_id`, repeated entity mentions, source-sentence
provenance, schema-compatible relation labels, and one/two-hop extracted KG
edges. Next best diagnostic is top-k evidence-path coverage before any new
training or augmentation.
